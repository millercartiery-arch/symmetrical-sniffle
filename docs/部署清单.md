# 部署清单（PM2 / 腾讯云服务器）

服务器路径：`/var/www/massmail`，域名：https://hkd.llc

---

## 一、本机构建

在项目根目录执行：

```powershell
cd "C:\Users\carti\OneDrive\Desktop\进行中\desktop-build-source"

# 前端
npm run build:frontend

# 后端
npm run build:backend
```

得到：
- `frontend/dist/`（含 `index.html`、`assets/`）
- `backend/dist/`（含 `index.js`、`routes/` 等）

---

## 二、上传到服务器

服务器**只接受 SSH 密钥**，不能用密码 scp。可用：

- **腾讯云文件管理**：在控制台打开该实例 → 文件管理 → 进入对应目录后上传并解压。
- **WinSCP / FileZilla**：用密钥登录后拖拽上传。

| 本机 | 服务器目标 |
|------|------------|
| `frontend/dist` 整份（或 zip 解压后） | `/var/www/massmail/frontend/dist/` |
| `backend/dist` 内所有文件 | `/var/www/massmail/backend/dist/` |
| `backend/ecosystem.config.cjs`（可选） | `/var/www/massmail/backend/ecosystem.config.cjs` |

若服务器上已有 `frontend/dist`，可只上传覆盖 `assets` 和 `index.html`；后端建议整份覆盖 `backend/dist`。

---

## 三、服务器上执行

SSH 登录后：

```bash
# 1. 若 dist 在 /root/frontend，可先拷到网站目录
cp -r /root/frontend/dist /var/www/massmail/frontend/   # 仅当需要时

# 2. 用 ecosystem 启动（推荐，会注入 FRONTEND_DIR）
cd /var/www/massmail/backend
pm2 delete massmail-api 2>/dev/null
pm2 start ecosystem.config.cjs
pm2 save

# 3. 看日志确认前端目录已加载
pm2 logs massmail-api --lines 10 --nostream
```

应看到：`[massmail-api] serving frontend from FRONTEND_DIR env: /var/www/massmail/frontend/dist`

---

## 四、验证

- 浏览器打开：https://hkd.llc ，按 **Ctrl+F5** 强刷。
- 若白屏：F12 → Network 看 `/assets/xxx.js` 是否 200；Console 看是否有报错。

---

## 五、仅更新前端或仅更新后端

| 只更新 | 本机 | 服务器 |
|--------|------|--------|
| **前端** | `npm run build:frontend` | 上传覆盖 `frontend/dist` → `pm2 restart massmail-api` |
| **后端** | `npm run build:backend` | 上传覆盖 `backend/dist` → `pm2 restart massmail-api` |

若使用 `ecosystem.config.cjs` 启动，`FRONTEND_DIR` 已在配置里，重启即可，无需再改环境变量。
