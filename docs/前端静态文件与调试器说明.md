# 前端静态文件与调试器说明

说明日志里常见的两类信息：**Node 调试器**（`Debugger listening on ws://...`）和 **FRONTEND_DIR not found**，以及在不同部署方式下的处理办法。

---

## 1. 为什么会看到 `Debugger listening on ws://0.0.0.0:9229/...`？

| 触发方式 | 解释 |
|----------|------|
| **`node --inspect` / `node --inspect-brk`** | Node 打开 V8 Inspector，监听 9229 端口，控制台会打印该行。 |
| **环境变量 `NODE_OPTIONS=--inspect`** | Dockerfile、.env 或 PM2 ecosystem 里设置后，每次启动都会开启调试。 |
| **PM2 的 `node_args: "--inspect=0.0.0.0:9229"`** | 在 ecosystem 里写死时，同上。 |

**影响**：开发时可远程用 Chrome DevTools 调试；生产环境若对外开放调试端口，有安全与资源占用风险。

**建议**：
- **生产**：去掉 `--inspect` 或改为 `--inspect=127.0.0.1:9229`（仅本机）。
- **PM2**：不需要调试时，在 `backend/ecosystem.config.cjs` 里删除或注释 `node_args: "--inspect=0.0.0.0:9229"`。

---

## 2. `[massmail-api] FRONTEND_DIR not found` 是什么？

后端启动时会查找**前端构建产物目录**（如 `frontend/dist`），用于托管静态页面与 `/assets/*`。若所有候选路径都不存在，就会打印：

```
[massmail-api] FRONTEND_DIR not found, tried:
  /var/www/massmail/frontend/dist,
  /var/www/massmail/backend/frontend/dist,
  ...
```

说明：
1. 服务器上**没有**这些路径，或路径下没有文件；
2. 若未设置环境变量 `FRONTEND_DIR`，代码会依次尝试上述路径，全部失败即报错。

**需要**：前端先 `npm run build` 生成 `dist`（或 `build`），再将该目录放到后端能读到的位置，并（推荐）通过 `FRONTEND_DIR` 明确指定。

---

## 3. PM2 部署时（无 Docker）

当前推荐方式：**直接用 PM2 在服务器上跑 Node**，不用 Docker。

### 3.1 前端产物

- 本机：`cd frontend && npm run build`，得到 `frontend/dist`（含 `index.html`、`assets/`）。
- 上传到服务器 **`/var/www/massmail/frontend/dist`**（网页上传、scp、或从 `/root/frontend/dist` 执行 `cp -r /root/frontend/dist /var/www/massmail/frontend/`）。

### 3.2 让后端认到前端目录：`FRONTEND_DIR`

在 PM2 下，**环境变量必须由 PM2 注入**，仅写进 `.env` 或当前 shell 的 `export` 可能不生效。推荐用 **ecosystem 配置文件** 写死 `FRONTEND_DIR`：

**`backend/ecosystem.config.cjs`**（项目内已提供示例）：

```js
module.exports = {
  apps: [{
    name: "massmail-api",
    cwd: "/var/www/massmail/backend",
    script: "dist/index.js",
    node_args: "--inspect=0.0.0.0:9229",  // 生产可删
    env: {
      NODE_ENV: "production",
      FRONTEND_DIR: "/var/www/massmail/frontend/dist",
    },
  }],
};
```

**在服务器上**：

```bash
cd /var/www/massmail/backend
pm2 delete massmail-api   # 若存在
pm2 start ecosystem.config.cjs
pm2 save
pm2 logs massmail-api --lines 15 --nostream
```

日志中应出现：**`[massmail-api] serving frontend from FRONTEND_DIR env: /var/www/massmail/frontend/dist`**，而不再出现 `FRONTEND_DIR not found`。

### 3.3 排查清单（PM2）

```bash
# 1. 确认前端目录存在且有权读
ls -la /var/www/massmail/frontend/dist/
ls /var/www/massmail/frontend/dist/assets/ | head -3

# 2. 确认后端代码支持 FRONTEND_DIR
grep -l "FRONTEND_DIR" /var/www/massmail/backend/dist/index.js

# 3. 用 ecosystem 启动并看日志
cd /var/www/massmail/backend
pm2 start ecosystem.config.cjs
pm2 logs massmail-api --lines 20 --nostream
```

## 4. Docker 部署时（可选参考）

若日后改用 **Docker** 或 **Docker Compose**，可参考以下思路。

### 4.1 多阶段构建（前端 dist 打进镜像）

- **阶段一**：在镜像里 `cd frontend && npm ci && npm run build`，得到 `frontend/dist`。
- **阶段二**：把 `frontend/dist` 拷贝进后端镜像的固定路径（如 `/app/frontend/dist`）。
- **运行**：`ENV FRONTEND_DIR=/app/frontend/dist`，且生产镜像 **不要** 设置 `NODE_OPTIONS=--inspect`。

### 4.2 卷挂载（开发时）

在 `docker-compose.yml` 中挂载宿主机已构建的 `frontend/dist`：

```yaml
volumes:
  - ./frontend/dist:/var/www/massmail/frontend/dist:ro
environment:
  - FRONTEND_DIR=/var/www/massmail/frontend/dist
```

### 4.3 调试器

- 生产：不设 `NODE_OPTIONS=--inspect`，或设为空。
- 开发：可设 `NODE_OPTIONS=--inspect=127.0.0.1:9229`，并映射端口 `9229:9229`。

## 5. 常见问题速查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| `FRONTEND_DIR not found` | 目录不存在或 PM2 未注入 `FRONTEND_DIR` | 确认 `dist` 在服务器上存在，并用 **ecosystem.config.cjs** 设置 `env.FRONTEND_DIR` 后重启 PM2。 |
| `/assets/xxx.js` 返回 HTML，白屏、MIME 错误 | 请求被 SPA 回退成 index.html | 确保后端先挂载 `/assets` 和静态目录（见 `backend/src/index.ts`），且使用带 `FRONTEND_DIR` 的新代码并重启。 |
| `Debugger listening` 一直在日志里 | 启动参数或 NODE_OPTIONS 带 `--inspect` | 生产环境去掉 `--inspect`；PM2 则删掉 ecosystem 里的 `node_args`。 |
| 权限 / EACCES | 目录属主或权限不对 | `chmod -R 755 /var/www/massmail/frontend/dist`，或检查运行 PM2 的用户是否有读权限。 |

---

## 6. 小结

- **调试器**：仅在需要时开启，生产关闭或绑定 127.0.0.1。
- **前端产物**：`npm run build` 得到 `dist`，放到服务器固定路径。
- **FRONTEND_DIR**：在 PM2 下用 **ecosystem.config.cjs** 的 `env.FRONTEND_DIR` 指定该路径，避免 “FRONTEND_DIR not found”。
- **校验**：`pm2 logs massmail-api` 中应出现 `serving frontend from FRONTEND_DIR env: ...`，且浏览器访问站点不再白屏或报 MIME 错误。
