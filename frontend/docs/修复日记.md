## 2026-03-17 依据桌面「修复文件」修正 proxy 外键

- **问题**：`proxy_audit.proxy_id` 与 `proxies.id` 类型不一致（前者 BIGINT UNSIGNED，后者 INT），导致外键约束报错 `Referencing column 'proxy_id' and referenced column 'id' in foreign key constraint 'fk_proxy_audit_proxy' are incompatible`。
- **修改**  
  - `backend/src/shared/proxy-schema.ts`：创建 `proxy_audit` 时改为 `proxy_id INT NULL`（与 `proxies.id` 一致）；外键添加失败时忽略 `ER_DUP_KEYNAME`、`ER_FK_DUP_NAME`、errno 1825；`account_proxy_bindings` 外键失败时同样忽略 `ER_DUP_KEYNAME`；为 `fk_proxy_audit_proxy` 增加 `ON UPDATE CASCADE`。  
  - 新增 `backend/scripts/migrate_proxy_audit_fk.sql`：供已有库将 `proxy_audit.proxy_id` 从 BIGINT UNSIGNED 改为 INT 并重新添加外键。
- **参考**：桌面「修复文件」中的类型统一与错误码说明。

### 依据桌面「修复文件2」（导入工具文件夹）的补充

- **错误码**：外键已存在时补充忽略 `ER_CANT_DROP_FK`（errno 1825），与文档一致。
- **索引**：proxy 三个索引改为 `CREATE INDEX IF NOT EXISTS`（MySQL 8.0.13+），避免重复创建报错。
- **日志**：添加外键或创建索引失败时输出 `console.error`，包含 `sqlMessage`、`code`、`stack`，便于排查。
- **类型策略**：当前项目保持 `proxies.id` 与 `proxy_audit.proxy_id` 均为 **INT**（与 schema_full_create 一致）；修复文件2 推荐的「两列均为 BIGINT UNSIGNED」可作为可选方案，若需采用需同步修改 account_proxy_bindings、sub_accounts 的 proxy_id 及迁移脚本。
- **复盘固化**：`backend/TROUBLESHOOTING.md` 已增加「proxy_audit 外键检查清单」（①–⑤，含 SQL 与期望），供 staging/生产部署前执行；并增加「可选：未来 INT→BIGINT UNSIGNED 迁移」表与思路，便于后续同事参考。CI 建议：typecheck 已通过；可选在 CI 中增加迁移脚本幂等测试（在 test 库执行 `migrate_proxy_audit_fk.sql` 两次）及 schema 一致性检查。
