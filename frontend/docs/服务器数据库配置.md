# 将数据库配置到服务器

在服务器 **43.160.225.156** 上让后端（API + Worker）使用正确的数据库与 Redis，需在服务器 **backend** 目录下配置 `.env`。

---

## 一、在服务器上创建/编辑 .env

1. **SSH 登录服务器**
   ```powershell
   ssh -i $env:USERPROFILE\.ssh\id_ed25519_new root@43.160.225.156
   ```
   或（项目根目录有 `.deploy_key` 时）：
   ```powershell
   ssh -i .deploy_key -o StrictHostKeyChecking=no root@43.160.225.156
   ```

2. **进入后端目录并编辑 .env**
   ```bash
   cd /var/www/massmail/backend
   nano .env
   ```
   若没有 `.env`，可先复制示例再改：
   ```bash
   cp .env.example .env
   nano .env
   ```

3. **填写数据库与 Redis（必填）**

   | 变量 | 说明 | 示例 |
   |------|------|------|
   | DB_HOST | MySQL 主机 | 本机用 `127.0.0.1` 或 `localhost`；远程库填其 IP/域名 |
   | DB_PORT | MySQL 端口 | `3306` |
   | DB_USER | 数据库用户名 | `massmail` 或你的用户名 |
   | DB_PASSWORD | 数据库密码 | 必填 |
   | DB_NAME | 数据库名 | `massmail` |
   | REDIS_URL | Redis 连接串 | `redis://localhost:6379` 或 `redis://:密码@host:6379` |
   | JWT_SECRET | 登录鉴权密钥 | 至少 32 位随机字符串 |
   | PORT | API 监听端口 | `3000`（与 Nginx 反代一致即可） |

   示例（数据库和 Redis 都在本机）：
   ```env
   DB_HOST=127.0.0.1
   DB_PORT=3306
   DB_USER=massmail
   DB_PASSWORD=你的MySQL密码
   DB_NAME=massmail
   REDIS_URL=redis://127.0.0.1:6379
   JWT_SECRET=请替换为至少32位随机字符串
   PORT=3000
   NODE_ENV=production
   ```

4. **保存并退出**  
   nano：`Ctrl+O` 回车保存，`Ctrl+X` 退出。

---

## 二、确保 MySQL 与 Redis 已安装并运行

在服务器上执行：

```bash
# MySQL
systemctl status mysql
# 或
systemctl status mysqld

# Redis
systemctl status redis
# 或
redis-cli ping
# 应返回 PONG
```

若未安装，按系统安装 MySQL / Redis 并启动服务；再创建库与用户（见下）。

---

## 三、创建数据库与用户（若尚未创建）

在服务器上登录 MySQL（或远程连到 DB_HOST 所在机器）：

```bash
mysql -u root -p
```

执行：

```sql
CREATE DATABASE IF NOT EXISTS massmail DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER IF NOT EXISTS 'massmail'@'%' IDENTIFIED BY '你的密码';
GRANT ALL ON massmail.* TO 'massmail'@'%';
FLUSH PRIVILEGES;
EXIT;
```

将 `你的密码` 换成与 `.env` 中 `DB_PASSWORD` 一致；若用户已存在可只改密码：`ALTER USER 'massmail'@'%' IDENTIFIED BY '新密码';`。

---

## 四、执行建表脚本（首次部署或新库）

在服务器上，若数据库为空，需执行完整建表：

```bash
cd /var/www/massmail/backend
mysql -u massmail -p massmail < scripts/schema_full_create.sql
```

或登录 MySQL 后：

```sql
USE massmail;
SOURCE /var/www/massmail/backend/scripts/schema_full_create.sql;
```

若库已存在但缺列（例如旧库），可让应用自动迁移：重启 API 后访问相关接口，代码中的 `ensure*Schema` 会尝试补列；或按需执行 `scripts/migrate_accounts_tn_columns.sql` 等迁移脚本。

---

## 五、重启后端使配置生效

```bash
cd /var/www/massmail/backend
pm2 restart massmail-api massmail-worker --update-env
pm2 save
```

查看日志确认无报错：

```bash
pm2 logs massmail-api --lines 30
```

若有 `ER_ACCESS_DENIED_ERROR` 或 `connect ECONNREFUSED`，检查 `.env` 中 DB_* 与 REDIS_URL 是否正确，以及 MySQL/Redis 是否允许该用户从本机或当前主机连接。

---

## 六、验证数据库连接

在**服务器上**执行（会使用服务器当前目录的 `.env`）：

```bash
cd /var/www/massmail/backend
npx tsx scripts/check_db_and_api.ts
```

应看到「数据库: ✓」「API: ✓」。若只测 API（不测本机 DB），可在本机执行：

```powershell
cd backend
$env:API_BASE="https://hkd.llc/api"; npx tsx scripts/check_db_and_api.ts
```

---

## 七、小结

| 步骤 | 说明 |
|------|------|
| 1 | SSH 登录 43.160.225.156 |
| 2 | 在 `/var/www/massmail/backend` 下创建/编辑 `.env`，填写 DB_*、REDIS_URL、JWT_SECRET 等 |
| 3 | 确保 MySQL、Redis 已安装并运行 |
| 4 | 创建库 `massmail` 和对应用户（若尚未创建） |
| 5 | 新库执行 `schema_full_create.sql` |
| 6 | `pm2 restart massmail-api massmail-worker --update-env && pm2 save` |
| 7 | 在服务器上运行 `check_db_and_api.ts` 验证 |

本地项目中的 `backend/.env.example` 可作为服务器 `.env` 的模板参考；**切勿将带真实密码的 `.env` 提交到代码库**。
