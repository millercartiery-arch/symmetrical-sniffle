# 连接服务器终端调试项目

## 一、SSH 连接命令

在**项目根目录**或任意位置打开 PowerShell / 终端，执行其一：

**方式 A（与 publish 脚本一致）**
```powershell
ssh -i $env:USERPROFILE\.ssh\id_ed25519_new root@43.160.225.156
```

**方式 B（与 deploy 脚本一致，需项目根目录有 .deploy_key）**
```powershell
ssh -i .deploy_key -o StrictHostKeyChecking=no root@43.160.225.156
```

连上后即进入服务器终端，可执行下面调试命令。

---

## 二、服务器项目路径与常用命令

| 说明 | 命令 |
|------|------|
| 项目根目录 | `cd /var/www/massmail` |
| 后端目录 | `cd /var/www/massmail/backend` |
| 查看 PM2 进程 | `pm2 list` |
| 查看后端日志 | `pm2 logs massmail-api` 或 `pm2 logs` |
| 重启后端 | `cd /var/www/massmail/backend && pm2 restart massmail-api` 或 `pm2 restart all` |
| 停止/启动 | `pm2 stop massmail-api` / `pm2 start massmail-api` |
| 查看环境 | `cd /var/www/massmail/backend && cat .env`（若有） |
| 本机检查 API | `curl -s http://127.0.0.1:3000/api/health` |
| 更新依赖并重启 | `cd /var/www/massmail/backend && npm install --production && pm2 restart all` |

## 三、一键打开连接（可选）

在项目根目录创建 `connect-server.ps1`，内容可为：

```powershell
# 连接服务器终端
$Key = "$env:USERPROFILE\.ssh\id_ed25519_new"
if (Test-Path $Key) { ssh -i $Key root@43.160.225.156 }
elseif (Test-Path ".deploy_key") { ssh -i .deploy_key -o StrictHostKeyChecking=no root@43.160.225.156 }
else { Write-Host "未找到 .ssh\id_ed25519_new 或 .deploy_key" }
```

执行：`powershell -ExecutionPolicy Bypass -File ./connect-server.ps1`

## 四、远程调试（Cursor/VS Code 附加到服务器 Node）

在本地用 Cursor 断点调试运行在服务器上的后端，需要：**SSH 端口转发** + **服务器以调试模式运行** + **本地 Attach**。

### 4.1 服务器端：以后台调试模式运行后端

先 SSH 连上服务器（见上文），在服务器上执行其一：

**方式 A：临时用 PM2 以调试模式启动（推荐）**
```bash
cd /var/www/massmail/backend
pm2 delete massmail-api 2>/dev/null
pm2 start dist/index.js --name massmail-api --node-args="--inspect=0.0.0.0:9229"
pm2 save
```
说明：`--inspect=0.0.0.0:9229` 让 Node 在 9229 端口监听调试，便于本机通过 SSH 隧道连接。

**方式 B：不用 PM2，直接前台运行（调试完需 Ctrl+C 再恢复 PM2）**
```bash
cd /var/www/massmail/backend
node --inspect=0.0.0.0:9229 dist/index.js
```
调试结束后恢复常驻进程：`pm2 start dist/index.js --name massmail-api` 或按你原有方式启动。

### 4.2 本机：建立 SSH 端口转发

在 Cursor 里：

1. **终端 → 运行任务**（或 `Ctrl+Shift+P` → “Tasks: Run Task”）
2. 选择 **「SSH: 端口转发 9229（远程调试）」**

会建立隧道：本机 `localhost:9229` → 服务器 `localhost:9229`。保持该终端不关闭。

若不用任务，也可在项目根目录手动执行：
```powershell
ssh -N -L 9229:localhost:9229 -i $env:USERPROFILE\.ssh\id_ed25519_new root@43.160.225.156
```

### 4.3 本机：附加调试器

1. 打开 **运行和调试**（Ctrl+Shift+D）
2. 在配置下拉选 **「Attach to Node (remote server)」**
3. 按 **F5** 或点击绿色播放按钮

连接成功后即可在本地代码（`backend` 目录）打断点，请求打到服务器时会命中。

### 4.4 调试结束后恢复服务器

若用方式 A 临时改了 PM2，调试完可改回普通启动并重启：
```bash
cd /var/www/massmail/backend
pm2 delete massmail-api
pm2 start dist/index.js --name massmail-api
pm2 save
```
