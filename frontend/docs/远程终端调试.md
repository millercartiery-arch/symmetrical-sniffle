# 远程终端调试（Remote Node Debug）

在本地 VS Code/Cursor 中附加到**远程服务器**上运行的 Node 进程，进行断点调试。

---

## 一、前提

- 本机可 SSH 到服务器（已配置密钥或密码）。
- 服务器上 backend 代码路径与本地一致或已知（用于 source map 映射）。

---

## 二、步骤

### 1. 在远程服务器上以调试模式启动

在服务器上进入 backend 目录，用 `--inspect` 启动（默认监听 9229）：

```bash
# 仅 API
NODE_OPTIONS='--inspect' node dist/index.js

# 或同时调试 Worker（需开第二个端口，例如 9230）
NODE_OPTIONS='--inspect=0.0.0.0:9229' node dist/index.js
```

若使用 pm2，可临时为单实例开 inspect：

```bash
pm2 delete massmail-api   # 或你的进程名
cd /var/www/massmail/backend
NODE_OPTIONS='--inspect=0.0.0.0:9229' pm2 start dist/index.js --name massmail-api
```

确保服务器防火墙或安全组**放行 9229**（仅内网时可只允许本机访问，通过下面 SSH 转发到本地）。

### 2. 本机建立 SSH 端口转发

把远程的 9229 转到本机 9229，这样 VS Code 才能连上。

**方式 A：用 VS Code 任务（推荐）**

1. 终端 → 运行任务 → 选 **「SSH: 端口转发 9229（远程调试）」**。
2. 保持该终端不关，转发会一直有效。

**方式 B：命令行**

```bash
ssh -N -L 9229:localhost:9229 -o ServerAliveInterval=60 root@你的服务器IP
```

若使用密钥（与 `.vscode/tasks.json` 里一致）：

```bash
ssh -N -L 9229:localhost:9229 -i %USERPROFILE%\.ssh\id_ed25519_new -o ServerAliveInterval=60 root@43.160.225.156
```

### 3. 在 VS Code/Cursor 中附加

1. 打开 **运行和调试**（Ctrl+Shift+D）。
2. 任选其一：
   - **「Attach to Node (remote server)」**：需已手动运行「SSH: 端口转发 9229」任务并保持运行。
   - **「Attach to Node (remote) + SSH 端口转发」**：F5 时会先启动 SSH 隧道再附加（隧道建立需几秒，若附加失败可重试一次）。
3. 按 **F5** 开始附加。

若代码在服务器上的路径是 `/var/www/massmail/backend`，且本地仓库的 backend 在 `${workspaceFolder}/backend`，断点会正确对应；否则可在 launch.json 里改 `remoteRoot` / `localRoot`。

---

## 三、配置说明（.vscode）

| 文件 | 作用 |
|------|------|
| **launch.json** | 「Attach to Node (remote server)」：附加到本机 `localhost:9229`（即经 SSH 转发后的远程进程）；`remoteRoot` 为服务器上的 backend 路径。 |
| **tasks.json** | 「SSH: 端口转发 9229」：建立 `本机 9229 → 服务器 localhost:9229` 的隧道；服务器 IP、密钥路径可在该任务里改。 |

修改服务器 IP 或密钥：编辑 `.vscode/tasks.json` 里 SSH 任务的 `args`（`-i` 与 `root@IP`）。

---

## 四、常见问题

| 现象 | 处理 |
|------|------|
| 附加时提示连接被拒绝 | 先确认本机已运行「SSH: 端口转发 9229」且无报错；再在服务器上执行 `curl -s http://localhost:9229/json` 看是否有 inspect 输出。 |
| 断点不命中或错行 | 检查 `launch.json` 的 `localRoot` / `remoteRoot` 是否与服务器实际路径、本地 backend 路径一致；若服务器用 `dist/`，需有 source map。 |
| 服务器上 9229 被占用 | 换端口，例如 `--inspect=0.0.0.0:9230`，同时把 SSH 与 launch 的端口改为 9230。 |

---

## 五、小结

1. **服务器**：`NODE_OPTIONS='--inspect=0.0.0.0:9229'` 启动 API（或 pm2 同参数）。
2. **本机**：运行任务「SSH: 端口转发 9229」建立隧道。
3. **VS Code**：选择「Attach to Node (remote server)」并 F5 附加。

完成后即可在本地对远程 Node 进程下断点、单步、查看变量。
