# 部署重构后的前端到服务器

服务器目录：`/var/www/massmail`，后端已运行在 PM2（massmail-api 会托管 `../frontend/dist`）。

---

## 方式一：用脚本自动同步并构建（推荐，需 Git Bash / WSL / Linux）

在**项目根目录**执行（把 `43.160.225.156` 换成你的服务器 IP）：

```bash
DEPLOY_HOST=root@43.160.225.156 DEPLOY_PATH=/var/www/massmail bash scripts/deploy-frontend.sh
```

脚本会：

1. 用 `rsync` 把本地 `frontend/` 同步到服务器 ` /var/www/massmail/frontend/`（排除 `node_modules`、`dist`）
2. SSH 到服务器执行：`cd /var/www/massmail/frontend && npm ci && npm run build`
3. 执行 `pm2 restart all`

**要求：** 本机已安装 `rsync`、`ssh`，能登录服务器（免密或输入密码）。

---

## 方式二：本机先构建，再上传 dist（无需在服务器装 Node）

1. **本机**在项目根目录执行：

   ```bash
   npm run build:frontend
   ```

2. 把生成的 `frontend/dist` 整个目录上传到服务器的 `/var/www/massmail/frontend/dist`。

   例如用 **scp**（在 PowerShell 或 Git Bash 中，把路径改成你的本机路径）：

   ```bash
   scp -r frontend/dist root@43.160.225.156:/var/www/massmail/frontend/
   ```

   或用 **WinSCP / FileZilla** 将 `frontend/dist` 拖到服务器 `/var/www/massmail/frontend/`。

3. **服务器**上重启 API，使静态文件生效：

   ```bash
   pm2 restart massmail-api
   ```

---

## 方式三：手动上传源码后在服务器构建

1. 用 **WinSCP / FileZilla / scp** 将整个 `frontend` 目录（可排除 `node_modules`、`dist`）上传到服务器 `/var/www/massmail/frontend/`。

2. **服务器**上执行：

   ```bash
   cd /var/www/massmail/frontend
   npm ci
   npm run build
   pm2 restart all
   ```

---

## 验证

- 浏览器访问：`http://43.160.225.156/`（或你的域名），应看到重构后的前端（登录、看板、任务等）。
- 若 404，检查 `pm2 logs massmail-api` 中是否打印了 `serving frontend from ...` 及路径是否正确。
