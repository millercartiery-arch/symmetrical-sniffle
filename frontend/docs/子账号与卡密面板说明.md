# 子账号与卡密面板说明

子账号面板基于 **credentials / card_keys / sub_accounts** 三张表，与调度器、代理池打通。

---

## 一、数据模型（数据库）

| 表 | 说明 |
|------|------|
| **credentials** | 全局登录凭证（username, password_hash/password_cipher），不存卡密 |
| **card_keys** | 卡密（code_enc、type、status、max_use、use_count、valid_from、valid_to） |
| **sub_accounts** | 子账号（调度粒度），关联 credential_id、card_key_id、proxy_id，含 status、weight、region、enabled 等 |

建表与迁移由 `ensureCardCredentialSchema` 在首次请求子账号/卡密接口时自动执行（见 `backend/src/shared/card-credential-schema.ts`）。完整建表脚本见 `backend/scripts/schema_full_create.sql`。

---

## 二、API（后端）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/card/activate | 卡密激活：body 含 code、region?、weight?、proxy_id?；校验卡密后创建一条 sub_account，并更新 card_keys.use_count |
| GET | /api/sub-accounts | 子账号列表：分页（page、limit）、过滤（status、region、proxy_id、tenant_id） |
| PATCH | /api/sub-accounts/:id | 更新子账号：仅允许 status、weight、proxy_id、region、enabled |
| DELETE | /api/sub-accounts/:id | 删除子账号（物理删除） |

---

## 三、前端面板（账号管理 → 子账号管理）

- **卡密激活**：弹窗输入卡密、地区、权重、绑定代理（可选），提交调用 POST /api/card/activate，成功后刷新列表。
- **列表**：展示 ID、凭证账号、凭证类型、卡密类型/状态、地区、权重、代理、状态、启用、创建时间；支持按状态/地区/代理筛选与分页。
- **编辑**：行内「编辑」打开简单表单，可改状态、权重、地区、绑定代理、启用，提交 PATCH /api/sub-accounts/:id。
- **删除**：行内「删除」二次确认后调用 DELETE /api/sub-accounts/:id。

已移除的旧逻辑：原「批量创建子账号」（users 表）、「自动分发」配额（POST /sub-accounts/batch、POST /sub-accounts/distribute）及对应弹窗已删除，子账号仅通过卡密激活创建。

---

## 四、与调度器、代理的打通

- 调度器（scheduler）从 **sub_accounts** 中选取 status=ready、enabled=1、有 proxy_id 的子账号参与任务分配；子账号与 **proxies** 通过 proxy_id 关联。
- 子账号面板中「绑定代理」即设置 sub_accounts.proxy_id；代理列表来自 GET /api/proxies，与 IP 配置 Tab 一致。
