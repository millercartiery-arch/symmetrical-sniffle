# 前端部署与白屏排查

## 现象：白屏且控制台报「未能加载样式表」或 404

说明 **index.html 能打开，但 `/assets/` 下的 CSS/JS 请求失败**（常见为 404）。

## 原因与处理

### 0. index.html 缺少 React 挂载点与入口脚本（代码问题）

- **现象**：页面全白，控制台可能无报错，或只有「Cannot read properties of null (reading 'render')」等。
- **原因**：`frontend/index.html` 里没有 `<div id="root"></div>` 和 `<script type="module" src="/src/main.tsx"></script>`，构建出的页面不会加载 React，导致白屏。
- **处理**：确保 `index.html` 包含上述挂载点和入口脚本（由 Vite 在构建时替换为带 hash 的 JS），然后重新执行 `npm run build` 并完整部署 `dist`。

### 1. 未完整部署 dist（最常见）

- **原因**：只上传了 `index.html`，或服务器上 `assets` 目录被覆盖/缺失，导致 HTML 里引用的 `index-xxx.css` / `index-xxx.js` 不存在。
- **处理**：
  - 每次部署必须**整体**上传/覆盖 `frontend/dist` 目录，保证 **dist 与 dist/assets 一起**：
    - `dist/index.html`
    - `dist/assets/*.js`
    - `dist/assets/*.css`
  - 不要只更新 `index.html` 而不更新 `assets`；也不要只更新 `assets` 而保留旧的 `index.html`（文件名带 hash，必须成对更新）。

### 2. 服务器未正确提供 /assets/ 静态文件

- **原因**：Nginx（或其它 Web 服务器）没有把 `/assets/` 映射到前端的 `dist/assets` 目录，或 root/alias 配错。
- **处理**：保证站点根目录指向**包含 index.html 和 assets 目录**的那一层（即 `dist` 所在目录），例如：

**Nginx 示例（站点在根路径 `/`）：**

```nginx
server {
    listen 80;
    server_name www.hkd.llc hkd.llc;
    root /var/www/massmail/frontend/dist;   # 此处为 dist 的完整路径
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;    # SPA 路由回退到 index.html
    }

    location /assets/ {
        # 可选：显式指定静态目录，若 root 已正确可省略
        alias /var/www/massmail/frontend/dist/assets/;
    }
}
```

- 确认：在服务器上执行 `ls /var/www/massmail/frontend/dist/assets/` 能看到 `index-*.css`、`index-*.js` 等文件。
- 若部署在**子路径**（如 `https://www.hkd.llc/app/`），需在构建前设置 `VITE_BASE_PATH=/app/` 并重新构建，且 Nginx 的 `root`/`location` 要对应到该子路径。

### 3. 部署在子路径

- 若访问地址为 `https://www.hkd.llc/应用名/`，需在构建前设置环境变量并重新构建：
  - `VITE_BASE_PATH=/应用名/`
  - 然后执行 `npm run build`
- 构建后 `index.html` 中的资源会变为 `/应用名/assets/xxx`，Nginx 需以该子路径为 root 或正确配置 location。

## 快速自检

1. **F12 → 网络(Network)**，刷新页面，看是否有红色失败请求。
2. 若 `https://www.hkd.llc/assets/index-xxx.css` 或对应 `.js` 返回 **404**：
   - 到服务器上确认 `dist/assets/` 下是否存在该文件名（每次构建 hash 会变，以当前 dist 为准）。
   - 确认 Nginx 的 `root`（或 `alias`）指向包含 `index.html` 和 `assets` 的目录。
3. 重新**完整**部署一次：删除服务器上旧的 `dist` 内容，再上传当前本地构建的整份 `dist`（含 `assets`）。

## 构建与 base 路径

- 默认根路径部署：直接 `npm run build`，生成资源路径为 `/assets/xxx`。
- 子路径部署：在 `frontend/.env.production` 或构建前设置 `VITE_BASE_PATH=/子路径/`，再执行 `npm run build`。
