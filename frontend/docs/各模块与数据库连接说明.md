# 各模块与数据库连接说明

所有后端模块均通过 **唯一连接池** `backend/src/shared/db.ts` 的 `pool` 访问 MySQL，无独立 createConnection/createPool。连接参数来自环境变量：`DB_HOST`、`DB_PORT`、`DB_USER`、`DB_PASSWORD`、`DB_NAME`。

---

## 一、连接入口

| 文件 | 说明 |
|------|------|
| **shared/db.ts** | 导出 `pool`（mysql2 createPool），默认 `connectionLimit: 50`，`DB_NAME=massmail`。 |

---

## 二、按模块使用的表

| 模块 | 使用的表 | 说明 |
|------|----------|------|
| **routes/account.ts** | users, accounts, audit_logs | 账号 CRUD、子账号(users)、锁定/探测/禁用、审计日志；GET /accounts（含 ?format=tn 与 message_tasks 统计）、POST /tn-accounts/import。 |

| **routes/inbound.ts** | accounts, message_tasks, contacts | 入站匹配账号、插入 Received 任务、更新 contacts。 |
| **routes/dashboard.ts** | message_tasks, accounts, campaigns, audit_logs | 仪表盘统计、活动汇总、任务状态分布；含 campaigns/message_tasks 的 schema 迁移。 |
| **routes/proxy.ts** | proxies, account_proxy_bindings | 代理 CRUD、绑定、拉取/白名单、探测状态。 |
| **routes/web-controller.ts** | proxies, account_proxy_bindings | 系统代理状态、子账号相关（若用 users 则同 account）。 |
| **workers/scheduler.ts** | message_tasks, campaigns, accounts, account_proxy_bindings, proxies | 取 Pending 任务、解析代理(bindings 优先，fallback accounts.proxy_url)、占锁、更新状态；含 message_tasks/accounts 的 schema 迁移。 |
| **workers/worker.ts** | message_tasks, accounts | 消费任务、更新任务状态与 error_msg、账号状态与 consecutive_errors；含 accounts 的 schema 迁移。 |
| **services/account.service.ts** | accounts | 账号状态更新、事件发布。 |
| **shared/proxy-schema.ts** | proxies, account_proxy_bindings | 建表与迁移。 |

## 三、表与主要字段（当前使用）

- **users**：id, username, password, role, quota_limit, api_key, tenant_id, updated_at
- **accounts**：id, phone, username, password, status, system_type, proxy_url( fallback ), tn_*, last_used_at, locked_by, locked_at, consecutive_errors, error_msg, rate_*, tenant_id
- **campaigns**：id, name, content, media_url, message_type, direction_mode, min_interval, max_interval, tn_account_ids, total_targets, status, tenant_id
- **message_tasks**：id, campaign_id, target_phone, content, media_url, message_type, direction_mode, status, account_id, locked_at, scheduled_at, retry_at, error_msg, error_code, created_at, updated_at, tenant_id（已从建表与迁移中移除多余列 error_message、processed_at、completed_at）
- **contacts**：phone, name, unread_count, last_activity, …
- **audit_logs**：user_id, action, details, tenant_id
- **proxies**：id, protocol, host, port, username, password, proxy_url_template, is_active, …
- **account_proxy_bindings**：account_id, proxy_id, session_key, is_primary, is_active

---

## 四、检查要点

1. **连接泄漏**：各模块均在 `try/finally` 中 `conn.release()`，或使用 `pool.query()` 不持有关键连接。
2. **单池**：除脚本（如 send-flow-check.mjs 自建 pool）外，仅 `shared/db.ts` 的 pool，便于限流与监控。
3. **环境变量**：部署与脚本需配置 `DB_HOST`、`DB_PORT`、`DB_USER`、`DB_PASSWORD`、`DB_NAME`，否则连接失败。
