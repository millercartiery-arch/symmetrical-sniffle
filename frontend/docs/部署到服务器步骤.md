# 部署项目到服务器步骤

目标服务器：**43.160.225.156**，路径：`/var/www/massmail`。

## 一、本地准备

1. **确认密钥**：项目根目录有 `.deploy_key`（SSH 私钥），且能连上服务器。
2. **确认配置**：
   - `backend/.env`：本地用于覆盖服务器（含 `ADMIN_PASSWORD_HASH` 等）；若希望保留服务器现有 `.env`，部署后需在服务器上改回或备份恢复。**服务器上数据库配置**详见 `docs/服务器数据库配置.md`，模板见 `backend/.env.example`。
   - `frontend/.env.production`：已配置 `VITE_API_BASE_URL=https://hkd.llc/api`（或你的域名）。

## 二、执行部署（在项目根目录 PowerShell）

```powershell
cd "c:\Users\carti\OneDrive\Desktop\进行中\desktop-build-source"
.\deploy.ps1
```

脚本会依次：

1. 构建 backend（`npm run build`）
2. 构建 frontend（`npm run build`）
3. SSH 创建远程目录 `backend`、`frontend/dist`
4. SCP 上传 `backend/*`、`frontend/dist/*` 到服务器
5. 在服务器执行：`npm install --production`、`pm2 restart massmail-api`、`pm2 restart massmail-worker`、`pm2 save`

若某步报错，根据提示修（如构建失败先修代码，SSH/SCP 失败检查 `.deploy_key` 与网络）。

## 三、部署后（可选）

- **PM2 环境变量**：若在服务器上改了 `backend/.env`，需执行  
  `pm2 restart all --update-env`
- **Nginx 入口不缓存**：若希望每次进入都是最新前端（避免“首次旧版、刷新新版”），在服务器 Nginx 里为 `index.html` 加不缓存头，详见 `docs/域名白屏排查.md` 第 6 节。
- **验证**：访问 https://hkd.llc（或你的域名），登录 admin / 你配置的密码；`pm2 list`、`pm2 logs massmail-api` 看进程与日志。

## 四、检查服务器上是否有前端文件

若不确定服务器是否已部署前端，可 SSH 登录后执行：

```bash
ls -la /var/www/massmail/frontend/dist/
ls -la /var/www/massmail/frontend/dist/assets/ 2>/dev/null || echo "assets 目录不存在"
```

- 若提示 `No such file or directory` 或 `assets` 为空，说明**没有前端文件**，需要按下面「首次/完整部署前端」执行一次。
- Nginx 的 `root` 需指向 `.../frontend/dist`，否则即使有文件也会 404 白屏。

## 五、仅上传前端（首次或更新）

**首次部署前端**（服务器上还没有 `frontend/dist` 时，先建目录再传）：

```powershell
# 本地构建
cd "c:\Users\carti\OneDrive\Desktop\进行中\desktop-build-source\frontend"
npm run build
cd ..

# 服务器上创建目录（若无）
ssh -i .deploy_key -o StrictHostKeyChecking=no root@43.160.225.156 "mkdir -p /var/www/massmail/frontend/dist"

# 上传整份 dist（含 index.html 和 assets）
scp -i .deploy_key -o StrictHostKeyChecking=no -r frontend/dist/* root@43.160.225.156:/var/www/massmail/frontend/dist/
```

若已有密钥在别处，把 `-i .deploy_key` 换成你的密钥路径；主机 43.160.225.156 换成你的服务器 IP。

**仅更新前端**（不改后端）时，同样执行上面「上传整份 dist」那一条即可。

## 六、仅上传后端并重启

若只改了后端：

```powershell
cd backend && npm run build && cd ..
scp -i .deploy_key -o StrictHostKeyChecking=no -r backend/* root@43.160.225.156:/var/www/massmail/backend/
ssh -i .deploy_key root@43.160.225.156 "cd /var/www/massmail/backend && npm install --production && pm2 restart massmail-api massmail-worker --update-env && pm2 save"
```
