# 查看应用面板「请求任务」和「导入账号」的数据

## 一、数据在哪里

| 功能 | 后端表 | 面板调用的 API（需登录 Bearer） |
|------|--------|--------------------------------|
| **请求任务**（活动 + 每条发送任务） | `campaigns`、`message_tasks` | `GET /api/user/campaigns`、`GET /api/user/tasks` 或 `GET /api/tasks` |
| **账号列表**（TN 协议号） | `accounts` | `GET /api/accounts`（分页）；`GET /api/accounts?format=tn`（TN 全量）；`POST /api/tn-accounts/import`（批量导入） |

面板请求这些接口时必须在请求头带 `Authorization: Bearer <token>`，否则会返回 401，面板就看不到数据。先保证登录成功、token 有效，面板才能正常显示。

---

## 二、直接在数据库里查看（不依赖面板）

连上 MySQL（本机或服务器上的 `massmail` 库），执行下面 SQL 即可查看当前数据。

### 1. 请求任务：活动列表（最近 20 条）

```sql
USE massmail;

SELECT 
  c.id,
  c.name,
  c.status,
  c.total_targets,
  c.created_at,
  (SELECT COUNT(*) FROM message_tasks t WHERE t.campaign_id = c.id AND t.status = 'Pending')   AS pending,
  (SELECT COUNT(*) FROM message_tasks t WHERE t.campaign_id = c.id AND t.status = 'Sent')     AS sent,
  (SELECT COUNT(*) FROM message_tasks t WHERE t.campaign_id = c.id AND t.status = 'Failed')  AS failed
FROM campaigns c
ORDER BY c.created_at DESC
LIMIT 20;
```

### 2. 请求任务：每条发送任务（最近 50 条）

```sql
SELECT 
  t.id,
  t.campaign_id,
  t.target_phone,
  t.status,
  t.error_msg,
  t.created_at,
  a.phone AS account_phone
FROM message_tasks t
LEFT JOIN accounts a ON t.account_id = a.id
ORDER BY t.created_at DESC
LIMIT 50;
```

### 3. 账号列表（原「导入账号」数据来源）

```sql
SELECT 
  id,
  phone,
  username,
  status,
  last_used_at,
  created_at,
  tenant_id
FROM accounts
ORDER BY updated_at DESC
LIMIT 100;
```

### 4. 汇总数量（一眼看有没有数据）

```sql
SELECT 
  (SELECT COUNT(*) FROM campaigns)     AS campaigns_count,
  (SELECT COUNT(*) FROM message_tasks)  AS message_tasks_count,
  (SELECT COUNT(*) FROM accounts)      AS accounts_count;
```

## 三、在面板里正常看到数据的前提

1. **登录成功**：用正确的管理员/操作员账号登录，拿到 token。  
2. **环境变量**：服务器上 `ADMIN_PASSWORD_HASH`、`JWT_SECRET` 已配置且一致。  
3. **前端 API 地址**：`VITE_API_BASE_URL` 指向当前后端，且带 `/api` 后缀（如 `https://你的域名/api`）。  
4. **多租户**：若启用了租户，请求会带 `tenantId`，数据库里 `campaigns`、`message_tasks`、`accounts` 的 `tenant_id` 需与当前用户一致，否则面板查不到数据。

按上面 SQL 能在库里看到数据，但面板仍为空时，多半是 1～3 中某一项没满足（尤其是 401 未授权）。
