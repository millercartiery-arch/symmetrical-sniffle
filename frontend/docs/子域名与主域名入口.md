# 子域名与主域名入口配置（可选）

当前默认使用单域名 **hkd.llc**（见 `frontend/.env.production`、`backend/.env`）。  
若需改为「主域名封装页 + 子域名应用」架构，可参考以下说明并替换为你的域名。

## 单域名（当前默认）

- **应用与 API**：`https://hkd.llc` / `https://hkd.llc/api`
- 无需子域名，Nginx 同一 server 即可提供前端与 API 反向代理。

## 若使用主域名 + 子域名

- **主域名**：封装入口页，可 302 或展示 `frontend/public/entry.html` 跳转到应用地址。
- **子域名**：应用工作区（SPA + API）。  
将下方示例中的 `app.cartier.us.cc` / `cartier.us.cc` 替换为你的子域名/主域名即可。

### Nginx 示例（子域名提供应用）

```nginx
server {
    server_name 你的子域名;  # 如 app.example.com
    root /var/www/massmail/frontend/dist;
    index index.html;
    location / { try_files $uri $uri/ /index.html; }
    location /api { proxy_pass http://127.0.0.1:3000; proxy_http_version 1.1; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_set_header X-Forwarded-Proto $scheme; }
    location /socket.io { proxy_pass http://127.0.0.1:3000; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection "upgrade"; }
    location /downloads { alias /var/www/massmail/backend/downloads; }
}
```

`backend/.env` 中 `ALLOWED_ORIGINS` 需包含该子域名（及主域名，若使用）。
