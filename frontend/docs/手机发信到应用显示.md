# 手机发消息到协议号在应用里显示

## 现象

用手机给协议号（如 +1 (574) 475-5511）发一条短信，希望应用 Chat 里能看到这条会话和消息。

## 原因说明

1. **应用左侧列表已改为「会话列表」**：显示的是「和你有过往来」的联系人（对方号码），不再只是协议账号列表。新收到的入站消息会出现在这个列表中。
2. **入站必须通过 webhook 写入**：应用不会主动去拉运营商/TextNow 的收件箱，需要有一个「入站推送」在收到短信时调用我们接口，把消息写入库。

## 实现「手机发到 5744755511 就在应用显示」的步骤

### 1. 入站 Webhook 接口

- **地址**：`POST /api/inbound/tn`
- **Body（JSON）**：
  - `phone`（必填）：**发件人**手机号（给你发短信的那台手机的号码）。
  - `content`（必填）：消息正文。
  - `to`（可选）：**接收方**号码，即我们的协议号（如 `5744755511` 或 `+1 (574) 475-5511`）。  
    若传了 `to`，会用该号在 `accounts` 里查对应账号并写入 `account_id` 和 `tenant_id`；不传则按「该发件人最近一条会话」推断账号。

示例：

```json
{
  "phone": "13800138000",
  "content": "你好",
  "to": "15744755511"
}
```

或：

```json
{
  "phone": "+1 (574) 475-5511",
  "content": "Hello",
  "to": "5744755511"
}
```

号码会做数字化处理（去空格、括号、横线等），只要与库里 `accounts.phone` 能对应即可。

### 2. 谁去调这个 Webhook？

- **TextNow / 虚拟号**：官方一般没有 HTTP 回调，需要你自己有一层「桥接」：
  - 例如用浏览器自动化（Playwright/Puppeteer）监听 TextNow 网页/App 的收件箱，收到新消息时请求 `POST /api/inbound/tn`。
  - 或使用支持 webhook 的短信网关（Twilio/Vonage 等），在网关里配置「收到短信时请求该 URL」。
- **自测**：用 Postman 或 curl 直接调一次，库里会有一条 `status='Received'` 的 `message_tasks` 和对应 `contacts`，应用里就会多一个会话并显示该条消息。

### 3. 应用侧行为

- 左侧列表来自接口：`GET /api/user/chat/conversations`（按租户过滤），列表项 = 对方号码 + 最后一条消息预览 + 未读等。
- 点击某个会话会请求：`GET /api/user/chat/messages?peerPhone=对方号码`，展示该会话的收发记录（含 `Received` 入站）。
- 会话列表每约 15 秒自动刷新一次，新入站会话会逐渐出现在左侧。

## 小结

- **应用无显示**通常是因为：**没有任何程序在「收到短信」时调用 `POST /api/inbound/tn`**。
- 已做的改动：
  - 入站接口支持 `to`，能正确关联协议号和租户；
  - Chat 左侧改为会话列表（对方号码），并带租户过滤和定期刷新；
  - 消息列表和会话列表都按 `tenant_id` 过滤，多租户下只看到本租户数据。

完成「收到短信 → 调用 /api/inbound/tn」的桥接后，手机发到 +1 (574) 475-5511 的消息就会在应用里显示。
