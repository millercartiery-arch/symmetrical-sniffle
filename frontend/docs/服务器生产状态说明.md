# 服务器配置与生产状态

## 一、配置检查结果（已确认）

| 项目 | 状态 | 说明 |
|------|------|------|
| **生产启动命令** | ✅ | 根目录：`npm run start:prod` → 调用 backend 的 `start:prod` |
| **NODE_ENV** | ✅ | `backend/package.json` 中 `start:prod` 使用 `cross-env NODE_ENV=production node dist/index.js`，启动时强制为 production |
| **后端入口** | ✅ | `backend/src/index.ts` 读取 `process.env.NODE_ENV`，未设时默认 development；用 start:prod 即为 production |
| **CORS** | ✅ | 仅当 `NODE_ENV === "production"` 时严格校验 origin，开发时放行 |
| **生产环境模板** | ✅ | `backend/.env.production.example` 含 DB、Redis、PORT、CORS_ORIGINS、JWT_SECRET 等说明 |

## 二、如何以生产状态运行

1. **构建**（若未构建过）  
   ```bash
   cd backend && npm run build
   ```

2. **环境变量**  
   - 复制 `backend/.env.production.example` 为 `backend/.env`（或部署机上的 .env），按实际填写 DB、Redis、JWT_SECRET、CORS_ORIGINS 等。  
   - 不设 `NODE_ENV` 也可，**只要用下面命令启动**，会由 cross-env 设为 production。

3. **启动（生产）**  
   - 在**项目根目录**：  
     ```bash
     npm run start:prod
     ```  
   - 或在 **backend 目录**：  
     ```bash
     npm run start:prod
     ```  
   - 进程内 `NODE_ENV=production`，后端以生产状态运行。

## 三、收尾退出

- 使用 **`npm run start:prod`** 启动即处于生产状态；无需在 .env 里再写 NODE_ENV。
- 关闭服务时直接停止该进程（Ctrl+C 或进程管理器）即可；下次启动仍用 `npm run start:prod` 确保生产状态。
