## 在服务器上执行（二选一）

### 方式 A：用 Node 脚本（需服务器上有 Node 和 backend 代码）

1. SSH 登录服务器：`ssh -i .deploy_key root@43.160.225.156`
2. 进入 backend 目录：`cd /var/www/massmail/backend`
3. 确保有 `.env` 且 `DB_HOST`/`DB_USER`/`DB_PASSWORD`/`DB_NAME` 指向**该服务器上的 MySQL**。
4. 执行：
   ```bash
   node scripts/run_reset_and_tenant.cjs
   ```

### 方式 B：用 MySQL 执行 SQL 文件

1. 在**本机**把 `backend/scripts/reset_stuck_tasks_and_locks.sql` 传到服务器，或直接在服务器上编辑该文件。
2. 在服务器上执行（把 `用户`/`密码` 换成实际 MySQL 账号）：
   ```bash
   mysql -u 用户 -p massmail < /var/www/massmail/backend/scripts/reset_stuck_tasks_and_locks.sql
   ```
   或登录 MySQL 后：`source /var/www/massmail/backend/scripts/reset_stuck_tasks_and_locks.sql;`

执行内容包含：安全释放（超 5 分钟锁） + **强制释放**（所有 LOCKED/Processing → Pending，所有 Busy → Ready） + **租户对齐**（`message_tasks` 与 `campaigns` 的 `tenant_id` 统一为 `subop_20260316_1` 的 `tenant_id`）。

## 确认环境一致性：JWT_SECRET

- 后端用 **.env 里的 JWT_SECRET** 签发和校验 JWT；重启后若 **.env 未改**，则 secret 不变，前端已拿到的 Bearer 仍有效。
- 你本机 backend `.env` 里当前是：`JWT_SECRET=dev-only-change-me-32-char-secret`。
- **请确认**：服务器上跑的后端（PM2）使用的 `.env` 中 **JWT_SECRET 与本机一致**（若本机前端连的是服务器 API，则服务器上的 JWT_SECRET 必须与签发该 token 时一致）。若曾改过服务器上的 JWT_SECRET 或未配置，会导致 401，需重新登录再刷新面板。
