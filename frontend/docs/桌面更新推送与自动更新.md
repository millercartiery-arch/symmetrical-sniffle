# 桌面应用：推送到服务器与应用内自动更新

## 一、推送更新到服务器（实现可更新）

### 1. 前置条件

- **Tauri 更新私钥**：已生成并放在 `%USERPROFILE%\.tauri\cartier-miller-updater.key`（与 `tauri.conf.json` 中 `updater.pubkey` 对应）。
- **SSH**：本机可用 `ssh -i %USERPROFILE%\.ssh\id_ed25519_new root@43.160.225.156` 登录服务器。
- **服务器目录**：`/var/www/massmail/backend/downloads/updates` 存在且可通过 HTTPS 访问为 `https://hkd.llc/downloads/updates/`。

### 2. 一键推送（推荐）

在项目根目录执行：

```powershell
npm run desktop:push
```

该命令会依次执行：

1. **构建**：`build-updater-artifacts.ps1` — 使用私钥构建带签名的 **updater + NSIS** 安装包（生成 .zip、.sig、.exe）。
2. **发布**：`publish-updater-to-server.ps1` — 生成 `latest.json` 并 SCP 上传 .zip、.sig、latest.json 到服务器。

### 3. 分步执行

- 仅构建（不推送）：`npm run desktop:build:updater`
- 仅推送（需已构建）：`npm run desktop:publish:updater`

### 4. 服务器端

- 更新清单地址：`https://hkd.llc/downloads/updates/latest.json`
- 桌面应用通过 `tauri.conf.json` 中 `updater.endpoints` 请求该 URL 判断是否有新版本。

## 二、应用内提示可更新 + 点击自动更新

### 1. 启动时检测并弹窗

- 应用启动约 1.5 秒后，在 Tauri 环境下会自动调用 `checkDesktopUpdate()`。
- 若服务器 `latest.json` 中版本号大于当前应用版本，会弹出 **「发现新版本」** 的 Modal，展示新版本号与说明。

### 2. 一键安装

- **弹窗内**：点击 **「立即更新」** 会执行 `installDesktopUpdate()`：下载更新包 → 校验签名 → 启动安装程序（覆盖安装），并显示“正在下载…”“正在启动安装…”等状态。
- **个人中心/设置**：在「个人中心」或「设置」中的「自动更新」卡片里，点击「检查更新」后若有新版本，会显示「立即更新（覆盖安装）」按钮，同样为一键安装。

### 3. 涉及文件

| 位置 | 说明 |
|------|------|
| `frontend/src/components/UpdatePromptModal.tsx` | 启动时检测更新并弹窗，弹窗内「立即更新」一键安装 |
| `frontend/src/components/AutoUpdateCard.tsx` | 个人中心/设置中的检查更新 + 立即更新按钮 |
| `frontend/src/utils/desktop-updater.ts` | `checkDesktopUpdate`、`installDesktopUpdate`、`onDesktopUpdaterEvent` |
| `src-tauri/tauri.conf.json` | `updater.endpoints`、`updater.pubkey` |

用户只需在发布新版本后执行一次 `npm run desktop:push`，之后旧版用户打开应用即可看到更新提示并点击「立即更新」完成覆盖安装。
