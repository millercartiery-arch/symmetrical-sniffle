# 服务器代码大扫除说明

本文记录已做的清理项，以及部署到服务器时建议**不要**带上的内容，避免垃圾代码与敏感文件上线。

---

## 一、已删除/已修正（本次大扫除）

| 项 | 说明 |
|----|------|
| **backend/src/simulate_traffic.ts** | 已删除。仅用于本地造数，内含硬编码 DB 密码与加密密钥，且使用旧表结构（如 token_cipher）。正式测试请用 `node scripts/send-flow-check.mjs --create-test <手机号>`。 |
| **backend/scripts/worker.js** | 已删除。旧版单进程轮询 worker，已被 `backend/src/workers/worker.ts` + BullMQ 替代，保留会造成混淆。 |
| **backend/scripts/check-tasks.cjs** | 已修正。去掉硬编码密码回退，改为仅使用 `process.env.DB_PASSWORD`，避免泄露。 |

---

## 二、部署时不要带上（.gitignore 已包含）

- `node_modules/`：在服务器上执行 `npm ci` / `npm install` 生成。
- `dist/`：在服务器上执行 `npm run build` 生成。
- `.env`、`.env.local`：含密钥，仅本地或服务器本地配置，不提交仓库。
- `*.log`、`scheduler.log`、`worker.log`、`backend/logs/`：运行时日志，不提交。

---

## 三、脚本与数据文件建议

| 路径 | 建议 |
|------|------|
| **scripts/send-flow-check.mjs** | 保留，运维排查用。 |
| **scripts/schema_full_create.sql**、**migrate_*.sql**、**reset_stuck_tasks_and_locks.sql** 等 | 保留，部署/迁移时按需执行。 |
| **scripts/custom_data.txt** | 若含真实号码或敏感内容，不要提交或不要部署到公网；可加入 .gitignore 或仅在内部使用。 |
| **scripts/test_*.js / test_*.cjs**（如 test_send_x2.js、test_import_and_campaign.js） | 多为一次性本地测试，生产环境可不部署或放入「仅开发用」目录。 |
| **scripts/simulate_*.cjs / simulate_full_traffic.js** | 本地压测/造数用，生产无需部署。 |

---

## 四、后续建议

1. **统一配置**：所有脚本从 `process.env` 或 `.env` 读 DB/Redis，不再写死密码。
2. **定期清理**：新加的一次性脚本用完后可移到 `scripts/archive/` 或删除，避免堆积。
3. **CI 检查**：若有 CI，可加一步检查仓库中是否仍包含 `massmailPassword`、`1234567890123456` 等字符串，防止敏感信息回填。
